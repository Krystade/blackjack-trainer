# v2 Cycle 3 Implementation Plan — App-Wide Audio for Eyes-Free Car Use

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Phase A — everything important in the app is spoken and/or chimed, so a passenger barely needs to look. Phase B — the three drills gain true eyes-free modes (spoken prompt, blind fixed-zone tap answer, spoken confirmation) usable without ever looking at the phone. Per spec `docs/superpowers/specs/2026-07-16-v2-cycle3-audio-design.md` (THE authority; task requirements = its sections + this plan).

**Architecture:** A new `src/audio/` module with a hard split between **pure script builders** (`narrate.ts` — engine data → strings, 100% unit-testable, zero browser APIs) and a **thin speech wrapper** (`speech.ts` — the only file that touches `window.speechSynthesis` / `AudioContext`, deliberately not unit-tested). Screens call `speak(narrateX(...))`; they never build strings inline. Blind-tap geometry lives in `zones.ts` as pure math so it is unit-testable without a DOM.

**Tech Stack:** unchanged (Vite 6 + React 18 + TS strict + `verbatimModuleSyntax`, Vitest, Playwright). No new dependencies — Web Speech API and WebAudio are browser built-ins; no audio asset files.

## Global Constraints

- Spec sections override this plan; STOP and flag disagreements.
- **No new npm dependencies.** Chimes are WebAudio oscillators; speech is `window.speechSynthesis`.
- **Audio is opt-in:** `AudioSettings.enabled` defaults to `false`.
- **Every browser API is absence-guarded.** `typeof window === 'undefined'`, missing `speechSynthesis`, missing `AudioContext`, and missing `navigator.wakeLock` must all degrade to a silent no-op, never throw. Unit tests run in jsdom/node where these are absent.
- **`narrate.ts` must import nothing from `speech.ts`** and must not reference `window`. It is pure functions over engine types.
- **True counts are always spoken sign-correct:** `+5` → "plus five", `-3` → "minus three", `0` → "zero". Never "negative three", never a bare "-3".
- **Grading is unchanged.** Eyes-free modes emit the same `GradedEvent`s and update the same stats as the visual modes. Audio never alters game or drill logic.
- strict TS; full `npm test` + `npm run build` (+ `npm run e2e` where stated) green before every commit.
- Commits end with `Co-Authored-By: Claude Opus 4.8 <noreply@anthropic.com>`.
- **Parity:** with `audio.enabled === false` (the default), every existing unit test and all 27 existing e2e specs pass unchanged and the UI renders identically.

**Key design notes for the implementer:**
- `isE2eAudioMode()` is true when `typeof window !== 'undefined' && window.location.search.includes('e2e=1')`. In that mode `speak()` and `chime()` push to `window.__speechLog` (creating it if absent) and **do not** call the real APIs. Cards/chimes both go in one ordered log: speech pushes the raw text, chime pushes `` `chime:${kind}` ``.
- Declare the global in `src/audio/speech.ts`:
```ts
declare global {
  interface Window { __speechLog?: string[] }
}
```
- `speak()` with `interrupt: true` calls `window.speechSynthesis.cancel()` first (in e2e mode, pushes nothing extra — just the text).
- `chime()` uses a lazily-created module-level `AudioContext`; frequencies: good = 880 Hz, bad = 220 Hz, attention = 1320 Hz; 0.12 s gain-ramped sine so it doesn't click. Guard `window.AudioContext ?? (window as any).webkitAudioContext`; if neither, return.
- Everything is a no-op when the API is missing — **never throw**.

**Wording rules (exact — the e2e specs assert these strings):**
- Suits: `s`→"spades", `h`→"hearts", `d`→"diamonds", `c`→"clubs".
- Ranks spoken: `A`→"ace", `2..10`→"two".."ten", `J`→"jack", `Q`→"queen", `K`→"king".
- `narrateTc`: `0` → `"zero"`; positive → `"plus " + word`; negative → `"minus " + word`. Numbers beyond twenty may fall back to digits — counts never realistically exceed ±20, but do not crash.
- `narrateResult('win', 2)` → `"Win, plus two."`; `'lose', -1` → `"Lose, minus one."`; `'push', 0` → `"Push."`; `'blackjack', 1.5` → `"Blackjack! Plus one point five."`; `'surrender', -0.5` → `"Surrender, minus zero point five."`
- `narrateHandResult(0, 2, 'win', 2)` → `"Hand one: Win, plus two."` When `handCount === 1`, delegate to `narrateResult` (no "Hand one:" prefix).
- `narrateCorrection(event)`: correct events → `"Correct."`; incorrect → `` `Wrong. ${event.reason} Correct play was ${narrateAction(expected)}. True count was ${narrateTc(event.tc)}.` `` — reason comes straight from the engine (it is already a human label such as `"16 v 10: stand at TC ≥ 0"`; do NOT re-derive it).
- `narrateBotAction('P2', 'hit', card)` → `"Player two hits, ten of clubs."`; without a card → `"Player two stands."` Map the seat label `P1..P5` to "player one".."player five".

### Task 8: Five-zone blind input component + hit-test math (spec "Phase B", second bullet; "Testing")

**Files:**
- Create: `src/audio/zones.ts`, `src/audio/zones.test.ts`, `src/ui/components/ZonePad.tsx`
- Modify: `src/audio/index.ts`, `src/ui/app.css`

**Interfaces:**
```ts
// src/audio/zones.ts  — PURE math, no DOM
export type ZoneId = 'hit' | 'stand' | 'double' | 'split' | 'surrender';
export type ZoneMode = 'action' | 'insurance';
/** Center circle wins over the quadrants; radius = 0.22 * min(w, h). */
export function hitTestZone(x: number, y: number, w: number, h: number, mode?: ZoneMode): ZoneId | 'take' | 'decline';
export const ZONE_LABEL: Record<ZoneId, string>; // 'hit' -> 'Hit', ...
```

**Layout (fixed, never moves):** top-left `hit`, top-right `stand`, bottom-left `double`, bottom-right `split`, center circle `surrender`. In `mode: 'insurance'`: left half `take`, right half `decline` (no center circle).

```tsx
// src/ui/components/ZonePad.tsx
export interface ZonePadProps {
  mode: ZoneMode;
  onAnswer: (zone: ZoneId | 'take' | 'decline') => void;
  onRepeat: () => void;   // fired by a long-press (>=600ms) anywhere
  visible: boolean;       // false => transparent but still tappable (true eyes-free use)
}
export function ZonePad(props: ZonePadProps): JSX.Element;
```
The component reads the tap coordinates from the pointer event relative to its own bounding rect and delegates every decision to `hitTestZone` — it must contain **no geometry of its own**, so the math stays unit-tested.

// src/audio/zones.test.ts
import { describe, it, expect } from 'vitest';
import { hitTestZone } from './zones';

const W = 400, H = 800;

describe('hitTestZone — action mode quadrants', () => {
  it('maps each corner to its fixed action', () => {
    expect(hitTestZone(10, 10, W, H)).toBe('hit');          // top-left
    expect(hitTestZone(390, 10, W, H)).toBe('stand');       // top-right
    expect(hitTestZone(10, 790, W, H)).toBe('double');      // bottom-left
    expect(hitTestZone(390, 790, W, H)).toBe('split');      // bottom-right
  });
  it('maps the exact center to surrender', () => {
    expect(hitTestZone(200, 400, W, H)).toBe('surrender');
  });
  it('gives the center circle priority over the quadrant it sits in', () => {
    // radius = 0.22 * min(400,800) = 88; a point 80px above center is inside it
    expect(hitTestZone(200, 320, W, H)).toBe('surrender');
    // 100px above center is outside the circle -> quadrant rules apply
    expect(hitTestZone(199, 300, W, H)).toBe('hit');
  });
  it('is stable at the quadrant boundaries (no gap, no overlap)', () => {
    expect(hitTestZone(0, 0, W, H)).toBe('hit');
    expect(hitTestZone(W, H, W, H)).toBe('split');
  });

    expect(hitTestZone(390, 400, W, H, 'insurance')).toBe('decline');
    expect(hitTestZone(200, 400, W, H, 'insurance')).toBe('decline'); // center is not special here
  });
});
